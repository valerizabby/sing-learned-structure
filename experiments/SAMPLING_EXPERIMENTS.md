# Sampling Parameters — Эксперименты

## Контекст

Сгенерированный трек звучал как "хаотичный набор нот": ноты короткие (1 бит = 0.67s),
без мелодической связности, случайные по высоте. Проведён grid search параметров
сэмплирования без переобучения модели.

Все эксперименты используют:
- Модель: `3SING*` (`trained_transformer_original_less_struct_combined`)
- Сохранённый prefix: `outputs/my_run/prefix_roll.pt` (32 бита, 90 BPM)
- Сегмент: `intro:4,verse:8,chorus:8,verse:8,chorus:8,outro:4`

## Диагноз

Основная причина хаотичного звучания выявлена анализом `outputs/my_run/generated.mid`:

| Метрика | Значение | Вывод |
|---------|---------|-------|
| Медиана длительности | 0.67s | Каждая нота = 1 бит, нет легато |
| pitch_std | 23.6 | Ноты по всему диапазону |
| Повтор той же ноты | 0% | Нет мелодической линии |

`topk_batch_sample` с k=50 и temperature=1.5 даёт почти равномерный softmax
по 50 нотам → 3 случайные независимые выборки → нет предсказуемости между шагами.

## Результаты

| Конфигурация | Нот | Медиана длит. | pitch_std | Оценка |
|---|---|---|---|---|
| Дефолт: k=50, s=3, t=1.5, ext=0 | 477 | 0.67s | 23.6 | Стаккато, хаос |
| k=5, s=3, t=1.0, ext=1 | 233 | 2.67s | 23.1 | **Коллапс на 6 нотах** |
| k=10, s=3, t=1.0, ext=0 | 584 | 0.67s | 20.5 | Лучше но всё ещё коротко |
| k=10, s=3, t=1.0, ext=2 | 356 | 2.67s | 23.4 | Слишком плотно |
| k=20, s=1, t=1.0, ext=1 | 301 | 1.33s | 21.8 | Слабовато |
| k=20, s=1, t=1.0, ext=2 | 269 | 2.00s | 23.5 | Слишком разброс |
| **k=10, s=1, t=1.0, ext=2** | **256** | **2.00s** | **19.5** | **✓ Лучший** |

## Выводы

1. **`note_extension` — главный параметр.** Без него медиана 0.67s (стаккато),
   с `ext=2` — 2.00s (3 бита). Это устраняет основной дефект звучания.

2. **k=5 → коллапс.** Слишком мало кандидатов, модель всегда выбирает одни и те же ноты.
   k=10 — минимум без коллапса.

3. **n_samples=1 лучше n_samples=3** при низком k — меньше плотность кластеров,
   ниже pitch_std.

4. **Понижение temperature (1.5→1.0) помогает** но недостаточно само по себе —
   без `note_extension` ноты всё равно остаются 1-битовыми.

5. **pitch_std ≈ 19–23 не устраняется параметрами сэмплирования** — это свойство
   модели, обученной генерировать ноты по всему диапазону фортепиано независимо.

## Лучшая конфигурация

```
--temperature 1.0 --temp_start 1.5 --temp_warmup_steps 10
--topk_k 10 --n_samples 1 --note_extension 2
```

Артефакт: `outputs/exp_D/generated.mid`

## Ограничения (не устраняются без переобучения)

- Широкий разброс по высоте нот (pitch_std ~20) — модель генерирует по всем регистрам
- Нет выраженной мелодической линии — авторегрессивная LSTM предсказывает каждый бит
  независимо, без явного голосоведения
- OOD-влияние text2midi prefix на первые шаги генерации
