### Слайд 6 — Модель 3SING* [5:30–7:00]

**Ключевое сообщение:** LSTM ловит локальную связность нот, StructureTransformer — глобальный план.

**Показать:**

*Архитектурная схема:*

```
SSM [B, T, T]
     ↓
StructureTransformer
  Conv1d(1→128) per row → TransformerEncoder(L=2, H=4)
     ↓ struct [T, B, 128]
         ⊕ × 0.05
LSTM output [1, B, 128]
     ↓ mixed
_apply_attention (sparsemax по SSM)
     ↓
output [128]
```

*Формула лосса:*
```
L = BCE + SSM_L2 + 0.03 × structure_loss
```

**Говорить:**  
«LSTM обрабатывает локальный контекст, StructureTransformer кодирует каждую строку SSM-матрицы через свёрточный энкодер и трансформер. Их выходы смешиваются с фиксированным коэффициентом α=0.05 — структурный трансформер выступает мягким регуляризатором».

## Резервные слайды (на вопросы)

### B1 — Как устроен SSM (определение)
- Формула: `SSM[i,j] = cosine_sim(chroma_i, chroma_j)`
- Визуал: пример chroma → SSM

### B2 — Почему не нейросеть для Seg2SSM?
- SALAMI = 422 трека, 7 меток → информации мало для нейросети
- Аналитика интерпретируема (объяснимо на одном слайде комиссии)
- A_empirical даёт верхнюю границу — её можно сравнить с A_fixed

### B3 — Почему LSA в 3× хуже по IOU?
- LSA: `attn = QK^T / √d · SSM` — SSM как маска уничтожает разреженное sparsemax routing
- ORIGINAL: `attn = sparsemax(SSM[t])` — каждый шаг выбирает из похожих тактов точно
- Вывод: для структурных паттернов критична разреженность, а не QKV

### B4 — Text2Prefix — ваша разработка?
- Нет. text2midi (amaai-lab) — внешняя модель на LLaMA
- Наш вклад: адаптер совместимости piano roll [T, 128], алайнмент tempo, prefix_in_sequence=False fix
- Модуль является системным компонентом, не научным вкладом

### B5 — Детали StructureTransformer
- Архитектура: `Conv1d(1, 128, k=3) → AdaptiveAvgPool1d(1) → TransformerEncoder(L=2, H=4)`
- Input: строка SSM `[T]` → `[128]` embedding
- Output: `[T, B, 128]` — позиционно-структурный контекст
- Параметров: ~200K; обучается с LSTM совместно
