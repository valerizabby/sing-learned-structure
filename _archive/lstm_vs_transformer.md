# Решение: не заменять LSTM на трансформер

**Дата:** 2026-04-18  
**Статус:** принято

## Контекст

Рассматривалась замена LSTM-ядра в `MusicGenerator` на трансформерный энкодер.

## Решение: оставить LSTM

### 1. SSM уже решает главную проблему LSTM

Основной аргумент в пользу трансформера — долгосрочные зависимости. В SingLS это покрыто структурно:

- SSM `[T×T]` кодирует глобальное сходство всех тактов сразу
- LSA (`SingLS/model/lsa.py`) — это однослойный self-attention (QKV + softmax) поверх LSTM-выходов с SSM как маской
- Итог: `LSTM (локальный контекст) + SSM-guided attention (глобальная структура)` архитектурно эквивалентно одному слою трансформера с инжектированным структурным prior

### 2. Авторегрессивный цикл несовместим с трансформером без рефакторинга

Тренировочный цикл (`SingLS/trainer/train.py:234`) — покомандный:

```python
for i in range(batch.shape[1] - starter_notes):
    output, _ = self.generator.forward(next_element, ...)
```

LSTM передаёт `self.hidden` между шагами — O(1) на шаг. Трансформер без KV-кэша делает O(T²) на каждом шаге. Переписывание потребовало бы полного рефакторинга генерации и тренировки.

### 3. Размер датасета

2600 треков — недостаточно для обучения трансформера с нуля. LSTM с attention при таком объёме данных — обоснованный выбор, что подтверждают метрики (IOU=0.32 у лучшей модели).

### 4. Трансформер в архитектуре уже есть

`HierarchicalGenerator` включает `StructureTransformer` (`SingLS/model/structure_transformer.py`), работающий на уровне структурных эмбеддингов. Трансформер присутствует там, где нужен — на уровне глобального плана, а не локальной генерации нот.

### 5. Формат SSM несовместим с относительным вниманием трансформера

SSM хранится как `[T_stacked, T]` (см. `batch_SSM`). Интеграция в трансформерное relative attention (как в Music Transformer, Huang et al. 2018) потребовала бы нетривиального преобразования к `[B, H, T, T]`.

## Альтернативы при необходимости улучшить capacity

| Опция | Сложность | Ожидаемый эффект |
|---|---|---|
| `num_layers=2` в LSTM | 1 строка | умеренный прирост IOU |
| Multi-head attention в LSA | ~20 строк | лучший захват структуры |
| Улучшение качества SSM (текущий фокус) | уже реализовано | прямое влияние на IOU |
